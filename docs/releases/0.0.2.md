# Release 0.0.2

## Release date

2026-02-25

## Scope

This release packages a production-ready macOS bootstrap profile for daily engineering work with AI tooling.

## What changed

- Added full Kubernetes module:
  - `kubernetes-cli`, `helm`, `kustomize`, `kind`, `minikube`, `kubectx`, `k9s`
  - health checks integrated into module flow
- Added deterministic container context handling in bootstrap:
  - prefer `orbstack` when installed
  - fallback to `colima` when needed
  - explicit recovery commands in troubleshooting
- Expanded verification and troubleshooting docs for full-stack checks
  - `docs/verification.md`
  - `docs/maintenance.md`
  - `docs/troubleshooting.md`
  - `docs/macos-quickstart.md`
  - `docs/wiki/kubernetes.md`
- Added `0.0.2` release artifact and wired release process updates

## Operator commands

### Install profile (recommended)

```bash
./install-macos.sh --start-orbstack --with-stats --configure-stats
```

### Baseline install

```bash
./install-macos.sh
```

### Quick verification

```bash
zsh -ic 'node --version; bun --version; python3 --version; python3.14 --version; uv --version; rustc --version; cargo --version; java -version'
zsh -ic 'kubectl version --client=true; helm version --short; kind version; minikube version; kustomize version'
zsh -ic 'docker context ls; docker context show; command -v kubectx; command -v kubens; k9s version'
zsh -ic 'git config --global --get core.pager; git config --global --get pager.diff; git config --global --get pager.show'
```

## Known drift checks

- Re-running `./install-macos.sh` is safe and idempotent.
- Container context now prefers `orbstack` if available, else `colima`.
- Kubernetes checks are validated via module + docs health section.

## Rollback and cleanup

- Re-run baseline installer in dry-run first:

```bash
./install-macos.sh --dry-run
```

- Reconcile runtime state after install issues:

```bash
docker context use orbstack || docker context use colima
if command -v orbctl >/dev/null 2>&1; then orbctl status; fi
kubectl version --client=true
```
