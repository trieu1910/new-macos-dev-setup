# Discoverability Playbook

This playbook tracks the repository growth setup for search and star conversion.

## Naming

- Repository name targets high-intent search phrase:
  - `new-macos-dev-setup`

## Metadata

- Description includes key phrases:
  - macOS setup
  - optimization
  - new computer software install
  - one-command bootstrap
- Topics are curated for setup and tooling queries.
- Homepage points to Wiki for deep-dive content.

## Content strategy

- README includes:
  - quick outcome statement
  - social proof badges
  - one-command quick start
  - docs + wiki links
- In-repo wiki provides domain-separated pages:
  - [System Tools](wiki/system-tools.md)
  - [Containers](wiki/containers.md)
  - [Kubernetes](wiki/kubernetes.md)
  - [JavaScript and TypeScript](wiki/js-ts.md)
  - [Python](wiki/python.md)
  - [Rust](wiki/rust.md)
  - [Flutter and Mobile](wiki/flutter.md)
  - [Java](wiki/java.md)
  - [C and C++](wiki/cpp.md)
  - [AI CLI](wiki/ai-cli.md)
  - [Terminal UX](wiki/terminal-ux.md)
  - [Verification and maintenance](wiki/verification-and-maintenance.md)

## OSS trust signals

- `LICENSE`, `SECURITY.md`, `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`
- issue forms + PR template
- CI badge and reproducible checks
- changelog and release process docs

## Conversion levers

- Keep setup command simple and above the fold.
- Keep defaults explicit (`python3` and `node` policy).
- Keep troubleshooting linked directly from README.
- Use Discussions for setup showcases and feedback loops.
